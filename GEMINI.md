# Project Workflow Mandates

## Branch Management
- **New Features:** For every new feature request, always create a new branch from `main`.
- **Default Pushing:** Always push the newly created branch to the remote repository immediately.
- **Cleanup:** Once a feature is confirmed to work and the user indicates they want to move to a new feature, delete the old feature branch (both locally and remotely) before creating the next one.

## Implementation Standards
- Maintain the visual aesthetic and consistency of the Gentlemen's Dashboard.
- Use Vanilla CSS for styling as per design constraints.
